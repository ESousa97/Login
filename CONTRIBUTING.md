# Contribuições

Obrigado por considerar contribuir!

## Fluxo

1. Faça um fork do repositório.
2. Crie uma branch descritiva: `feat/sua-ideia`.
3. Execute lint e testes antes do PR:
   - `npm run lint`
   - `npm run test`
4. Abra o Pull Request descrevendo o que mudou.

## Padrões

- Respeite a estrutura do projeto.
- Atualize a documentação quando necessário.
- Mantenha o foco em melhorias incrementais e simples.
